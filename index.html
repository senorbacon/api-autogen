Autogen API

This RESTful API generates its own documentation and unit-test harness from its source code - not from comments! This prevents the hugely common problem of docs and tools getting out of sync with the API as it evolves.

Click on the nodes on the left hand side to get documentation and a unit test interface for that node. Each node has different verbs like GET, DELETE, etc., and each of these verbs can be executed using the unit test harness. The example below is just a prototype... executing the verbs just outputs the parameters passed in.

When you select a particular verb, the interface displays all of the parameters that can or must be included with the request. There is also a checkbox that prevents the interface from rejecting invalid input, so that errored or harmful input can be sent to the API.
